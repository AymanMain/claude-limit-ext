Privacy Policy

Claude Limit Tracker — Last updated: May 2026

Claude Limit Tracker is an unofficial browser extension and is not affiliated with Anthropic. This policy describes how the extension handles data on your device.

Summary

All data the extension reads or stores stays on your device. Nothing is sent to any server operated by this extension or any third party.

What the extension reads from claude.ai

To identify your Claude organization ID, the extension runs a content script on https://claude.ai/* pages. This script:

  • Scans the page URL, page scripts, and anchor links for UUID-shaped strings
  • Reads localStorage and sessionStorage on claude.ai pages and searches for UUID-shaped strings
  • Intercepts outgoing fetch calls made by the claude.ai app to extract organization IDs from request URLs

Only UUID strings matching the org ID pattern are retained. All other content (page text, storage values, request bodies, response bodies) is discarded immediately and never stored or transmitted.

What the extension requests from claude.ai

Once the organization ID is known, the extension periodically calls:

https://claude.ai/api/organizations/{orgId}/usage

This request uses your existing browser session (credentials: include) — the same session used when you are logged in to claude.ai. The extension does not inject, read, copy, or store any cookies or session tokens. It only reads the JSON usage data returned by the endpoint (usage percentages and reset timestamps).

What is stored locally

The extension stores the following in chrome.storage.local on your device:

  • Your Claude organization ID
  • Usage percentages (5-hour and 7-day windows)
  • Reset timestamps
  • Extension settings (sync interval, notification thresholds, badge mode)
  • Notification state (which alerts have already fired, snooze state)

This data never leaves your device.

What is never stored

  • Claude session cookies or authentication tokens
  • Prompts or conversation content
  • Model responses
  • Any personally identifiable information beyond the organization ID

Third-party data sharing

No data is sent to any third-party server. The extension makes no analytics calls, crash reporting requests, or telemetry requests of any kind.

Permissions used

  • storage — saves usage data and settings locally
  • alarms — schedules background refresh every 2–15 minutes
  • notifications — sends local usage alerts
  • tabs — queries open claude.ai tabs to forward the detect-org-ID command to the content script
  • host permission: https://claude.ai/* — required for the content script and usage API requests

Data removal

All stored data can be cleared from the extension settings page under Privacy → Clear all local data. Uninstalling the extension removes all stored data automatically.
